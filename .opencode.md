# TechnoRouge — Development Roadmap

> **Vision:** Slay the Spire meets Cyberpunk — a roguelike deckbuilder with deep class
> identity, neon-drenched atmosphere, and meaningful run-to-run progression.
> Think: navigating a corrupted megacorp's network, floor by floor, hacking your
> way through ICE, daemons, and firewalls — with each run feeling different because
> of your class, your deck, and your relics.

---

## Current State

A working single-combat prototype: 4 classes, 12 cards, 5 enemies, full combat
loop with energy/draw/block/poison/weak/vulnerable. DOM-based UI over a Phaser
canvas background. No progression, no rewards, no map, no sound, placeholder art.

---

## Class System & Card Unlocks

> *Each class is a distinct playstyle with its own starting deck. Cards unlock
> as you progress, giving players reasons to try new builds.*

### Class Starting Decks (provided at run start)

Each class comes with a **unique starting deck** of 10 cards, defining their
playstyle from floor 1. These are NOT shared between classes.

| Class | Starting Deck (10 cards) | Playstyle |
|-------|--------------------------|-----------|
| **Netrunner** | 3 Strike, 3 Defend, 1 Inject, 1 Barrier, 1 Overclock, 1 Turbo | Balanced — strong fundamentals, versatile |
| **Cracker** | 5 Strike, 2 Defend, 1 Inject, 1 Neural Blitz, 1 Overload | Aggressive — fast damage, glass cannon |
| **Guardian** | 2 Strike, 4 Defend, 2 Barrier, 1 Fortify, 1 Overclock | Defensive — walls of block, outlast |
| **Infectant** | 2 Strike, 2 Defend, 2 Worm, 1 Data Leak, 1 Overclock, 1 Turbo, 1 Overload | DoT — poison and debuff, whittle them down |

### Card Unlock System

- [ ] **Starter cards are always available** — no unlock needed for the base pool
- [ ] **Unlock new class-specific cards** by completing milestones:
  - Beat Act 1 → unlock Act 1 class cards
  - Beat Act 2 → unlock Act 2 class cards
  - Beat Act 3 → unlock Act 3 class cards
  - Complete specific achievements → unlock special/secret cards
- [ ] **New cards enter the reward pool** — they appear in post-combat card choices
- [ ] Unlockable cards are **class-locked** — only appear for the relevant class
- [ ] Shared neutral cards unlock globally (available to all classes)
- [ ] Track unlocks in localStorage, show a "Collection" screen on main menu
- [ ] Each class ultimately has **~15-20 unique cards** + access to ~20 neutral cards

## Phase 1 — The Roguelike Loop (Core StS DNA)

> *This is the single most important phase. Without a map and progression, it's a
> tech demo, not a game.*

### 1a. Map / Network Topology
- [ ] Generate a branching path of nodes (like StS's map) themed as **network nodes**
- [ ] Node types: `Combat`, `Elite`, `Shop`, `Rest Site`, `Event`, `Boss`
- [ ] Player picks a path forward each floor (visible branching choices)
- [ ] ~15 floors per act, 3 acts per run
- [ ] Act 1: "Corporate Intranet" — basic ICE, daemons
- [ ] Act 2: "Deep Network" — tougher enemies, new mechanics
- [ ] Act 3: "Core Mainframe" — bosses, final challenge

### 1b. Encounter System
- [ ] Multiple enemy pool per act (scale from Act 1 → 3)
- [ ] Elite encounters: tougher enemies with special rewards (rare cards, relics)
- [ ] Boss encounters: 3 unique bosses (one per act), large HP pools, multi-phase or unique mechanics
- [ ] Enemy scaling: HP and damage scale slightly with floor number

### 1c. Post-Combat Rewards
- [ ] **Card Reward Screen** — choose 1 of 3 random cards to add to deck (after every combat)
- [ ] **Skip** option (don't add a card)
- [ ] **CryptoBytes** — earn CryptoBytes from combat (more for elites/bosses)
- [ ] **Ram Chip reward** — consumable items (Phase 2)

## Phase 2 — Economy & Shops

> *Give the player meaningful choices about spending resources.*

### 2a. CryptoByte System
- [ ] Earn CryptoBytes from combat (base + floor scaling + elite bonus)
- [ ] Spend CryptoBytes at shops
- [ ] CryptoBytes carry across the run, lost on death
- [ ] Currency UI: stylized as a glowing hex-code ticker in the HUD

### 2b. Shop Nodes
- [ ] Buy cards (3 random, priced by rarity)
- [ ] Remove a card from your deck (escalating cost: 50/75/100 CryptoBytes)
- [ ] Buy relics (Phase 3)
- [ ] Buy consumables (Phase 2c)

### 2c. Consumables / Ram Chips
- [ ] 4-6 consumable types (one-time use, held in a small inventory)
- [ ] Examples:
  - **Stim Pack** — heal 15 HP
  - **Overclocker** — gain 2 energy this turn (usable outside your turn?)
  - **Defrag** — gain 10 block
  - **Zip Drive** — draw 3 cards
  - **Virus Bomb** — deal 10 damage to all enemies
- [ ] Can hold max 3 at a time

## Phase 3 — Relics (Passive Modifiers)

> *The secret sauce of StS. Relics define builds and create "oh yes!" moments.*

- [ ] Relic inventory UI (top of screen, small icons)
- [ ] Relic categories:
  - **Common** — small passive bonuses (e.g., "+1 max energy", "heal 2 HP at start of combat")
  - **Uncommon** — build-around effects (e.g., "first card each turn costs 0", "whenever you play a virus card, apply 1 poison to all enemies")
  - **Rare** — powerful, run-defining (e.g., "start each combat with 3 strength", "when you discard a card, deal 3 damage to a random enemy")
  - **Boss** — guaranteed from boss kills, always strong
  - **Shop** — purchasable relics in shops
- [ ] ~30-40 relics total across all categories
- [ ] Relic synergy with the 4 classes (some relics better for certain builds)

## Phase 4 — Content Expansion

> *More cards, enemies, events. The game needs depth to stay interesting.*

### 4a. Card Pool Expansion
- [ ] Expand from 12 cards to **60-80 cards** across all types
- [ ] Each class gets **15-20 unique class-locked cards** (see Class System above)
  - Cards are gated by act progression (Act 1 unlocks, Act 2 unlocks, etc.)
  - Starter cards are always available; unlockable cards enter the reward pool
- [ ] Shared neutral pool (~20 cards any class can find in rewards)
- [ ] New card mechanics:
  - **Exhaust** — card is removed from deck for the rest of combat
  - **Ethereal** — discarded at end of turn if not played
  - **Retain** — kept in hand between turns (not discarded)
  - **Innate** — always in opening hand
  - **X-cost** — spend all remaining energy, effect scales
  - **Channel/Invoke** — summon a daemon program that acts each turn (new mechanic)
- [ ] Card upgrades: after elite kills or at rest sites, upgrade a card (stronger version)

### 4b. Enemy & Boss Expansion
- [ ] Expand enemy pool to **15-20 enemies** across acts
- [ ] 3 bosses (one per act), each with unique mechanics:
  - **Act 1 Boss: "Firewall Overseer"** — multi-phase, shields allies, enrages at low HP
  - **Act 2 Boss: "Zero-Day Exploit"** — shuffles trash into your deck, adapts to your playstyle
  - **Act 3 Boss: "The Core AI"** — massive HP, multiple attack patterns, summons minions
- [ ] Enemy artifacts (already scaffolded) — some enemies start with debuff immunity or other bonuses

### 4c. Random Events
- [ ] 10-15 events spread across floors
- [ ] Offer meaningful risk/reward choices
- [ ] Examples:
  - **"Corrupted Terminal"** — lose 10 HP, gain a rare card
  - **"Black Market Dealer"** — pay 50 CryptoBytes, remove a card and gain a relic
  - **"Memory Fragment"** — choose: heal 20 HP OR gain 100 CryptoBytes
  - **"Virus Upload"** — add 2 random virus cards to your deck (some good, some bad)
  - **"System Defrag"** — upgrade a random card
- [ ] Events should feel narrative, with cyberpunk flavor text

### 4d. Rest Sites
- [ ] **Heal** — restore 30% max HP
- [ ] **Upgrade** — upgrade a card
- [ ] **Remove** — remove a card from deck
- [ ] **Rest** — small heal + minor buff for next combat

## Commit Instructions

> All work goes through feature branches → PR → merge into `pre-3d` (default).
> Never commit directly to `pre-3d` or `main`.

### Branch Naming
- `feat/<feature-name>` — new features
- `fix/<bug-name>` — bug fixes
- `refactor/<what>` — code cleanup without behavior change
- `chore/<what>` — tooling, config, non-code changes

### PR Workflow
1. Create feature branch from `pre-3d`
2. Make changes, commit with descriptive message
3. Push branch, create PR targeting `pre-3d`
4. PR must pass lint (`npm run lint`) and build (`npm run build`) with 0 errors
5. Merge PR after review

### Commit Message Format
```
<type>: <short description>

<body — what changed and why>
```

Types: `feat`, `fix`, `chore`, `refactor`, `docs`, `test`

### Examples
```
feat: add map/network topology system (#30)

- Branching node graph with 6 node types
- Procedural generation with weighted distribution
- MapScene with visual rendering and node selection
- CombatScene now accepts enemyType from map
```

```
chore: add ESLint config with TypeScript support

- eslint.config.js with typescript-eslint
- lint and lint:fix scripts in package.json
- Warnings for no-explicit-any and no-console
```
