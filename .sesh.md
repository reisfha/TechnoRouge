# Session Log — 2026-07-21

## Goal
Emulate the web Phaser game as a native Expo Go mobile app with feature parity and seeded RNG for reproducible runs.

## What We Built

### Shared Game Logic Package (`lib/game-logic/`)
Created `@workspace/game-logic` — a pnpm workspace package that both the web and mobile projects import from, eliminating code duplication.

**Files created:**
- `lib/game-logic/package.json` — workspace package manifest
- `lib/game-logic/tsconfig.json` — extends root tsconfig
- `lib/game-logic/src/index.ts` — barrel exports
- `lib/game-logic/src/rng.ts` — `SeededRNG` class (mulberry32 algorithm)
- `lib/game-logic/src/Game.ts` — canonical GameState singleton (merged web + mobile features)
- `lib/game-logic/src/data/` — cards, classes, enemies, map
- `lib/game-logic/src/entities/` — Card, Player, Enemy, Effect
- `lib/game-logic/src/systems/` — DeckSystem, EffectSystem
- `lib/game-logic/src/utils/` — helpers (shuffleArray, generateId)

### Seeded RNG
Added `SeededRNG` class with:
- `next()` — returns 0..1
- `nextInt(min, max)` — integer in range
- `shuffle(array)` — Fisher-Yates shuffle

Integrated into:
- `Game.startRun(className, seed?)` — optional seed, auto-generates if omitted
- `generateMap(act, rng)` — deterministic map generation
- `DeckSystem.shuffleDeck(deck, rng)` — seeded card shuffling
- `Player.drawFromDiscard()` — uses rng
- `Enemy` constructor — HP variance uses rng
- `Game.spawnEnemies()` — enemy count and selection uses rng

`Game.seed` exposed for display in both UIs.
